运行容器

docker run 是启动容器的方法。通过前面的Dockerfile 时我们已经学习到，可用三种方式指定容器启动时执行的命令。

容器运行执行的命令

通过下面三种方式，我们来执行运行在容器中的命令。

  • CMD 指令
  • ENDPOINT 指令
  • 在 docker run 命令行中指定

运行一个容器示例

[root@docker ~]# docker run ubuntu pwd
/
[root@docker ~]# docker ps -a
CONTAINER ID        IMAGE                                                                      COMMAND                  CREATED              STATUS                          PORTS               NAMES
e12b964641c4        ubuntu                                                                     "pwd"                    32 seconds ago       Exited (0) 31 seconds ago                           modest_johnson

说明： 此处我们运行一个ubuntu镜像,并运行pwd命令后,容器便退出了,状态为Exited。这种“一闪而过”的容器通常不是我们想要的结果，我们希望容器能够保持 runing 状态，这样才能被我们使用。

让容器长久的运行

因为容器的生命周期依赖于启动时执行的命令，只要该命令不结束，容器也就不会退出。理解了这个原理，我们就可以通过执行一个长期运行的命令来保持容器的运行状态。

[root@docker ~]# docker run ubuntu /bin/bash -c "while true;do sleep 1;done"
[root@docker ~]# docker ps -a
CONTAINER ID        IMAGE                                                                      COMMAND                  CREATED             STATUS                     PORTS               NAMES
d36bbcefec6b        ubuntu                                                                     "/bin/bash -c 'while…"   50 seconds ago      Up 49 seconds                                  eager_lumiere

上面利用执行循环命令，让命令一直执行，容器自然就不会退出(Exited)状态,而是一直运行(Up)状态。但这样终端会被一直占用。可以利用加上参数-d 以后台方式运行容器。

[root@docker ~]# docker stop d36bbcefec6b
d36bbcefec6b
[root@docker ~]# docker rm d36bbcefec6b
d36bbcefec6b
[root@docker ~]# docker run -d ubuntu /bin/bash -c "while true;do sleep 1;done"
a141b831a16edbb94ab41553e2be2af79e2d5b68f25877db40750c49cc991649

[root@docker ~]# docker ps -a
CONTAINER ID        IMAGE                                                                      COMMAND                  CREATED             STATUS                      PORTS               NAMES
a141b831a16e        ubuntu

说明事项：

  • CONTAINER ID 是容器的 “短ID”，前面启动容器时返回的是 “长ID”。短ID是长ID的前12个字符。
  • NAMES 字段显示容器的名字，在启动容器时可以通过 –name 参数显示地为容器命名，如果不指定，docker 会自动为容器分配名字。
  • 我们可以利用docker stop CONTAINER ID 停止正在运行的容器，并使用docker rm CONTAINER ID 删除停止的容器;docker run -d COMMAND 以后台的方式运行容器。

进入容器内部

我们经常需要进到容器里去做一些工作，比如查看日志、调试、启动其他进程等。有两种方法进入容器：attach 和 exec。

docker attach

通过 docker attach 可以 attach 到容器启动命令的终端，例如：
[root@docker ~]# docker run -d ubuntu /bin/bash -c "while true;do sleep 1;echo 'I am in container';done"
 7b0dfb366bcb3259f6ec13e016e1f517b7c56cf0d406fdea4e4215e55cfd7e86

这次我们通过 “长ID” attach 到了容器的启动命令终端，之后看到的是echo 每隔一秒打印的信息。

[root@docker ~]# docker attach 7b0dfb366bcb3259f6ec13e016e1f517b7c56cf0d406fdea4e4215e55cfd7e86
I am in container
I am in container
I am in container
I am in container
I am in container

docker exec

通过 docker exec 进入相同的容器：

[root@docker ~]# docker ps
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS               NAMES
7b0dfb366bcb        ubuntu              "/bin/bash -c 'while…"   4 minutes ago       Up 4 minutes                            zealous_dubinsky
[root@docker ~]# docker exec -it 7b0dfb366bcb  bash
root@7b0dfb366bcb:/# ls
bin   dev  home  lib64  mnt  proc  run   srv  tmp  var
boot  etc  lib   media  opt  root  sbin  sys  usr
root@7b0dfb366bcb:/# ps -ef
UID         PID   PPID  C STIME TTY          TIME CMD
root          1      0  0 04:20 ?        00:00:00 /bin/bash -c while true;do sle
root        265      0  0 04:24 pts/0    00:00:00 bash
root        279      1  0 04:24 ?        00:00:00 sleep 1
root        280    265  0 04:24 pts/0    00:00:00 ps -ef

说明事项：

  • -it 以交互模式打开 pseudo-TTY，执行 bash，其结果就是打开了一个 bash 终端
  • 进入到容器中，容器的 hostname 就是其 “短ID”
  • 可以像在普通 Linux 中一样执行命令。ps -elf 显示了容器启动进程while 以及当前的 bash 进程
  • 执行 exit 退出容器，回到 docker host
  • docker exec -it bash|sh 是执行 exec 最常用的方式

attach 对比 exec

  • attach 直接进入容器 启动命令 的终端，不会启动新的进程。
  • exec 则是在容器中打开新的终端，并且可以启动新的进程。
  • 如果想直接在终端中查看启动命令的输出，用 attach；其他情况使用 exec。

当然，如果只是为了查看启动命令的输出，可以使用 docker logs 命令

[root@docker ~]# docker ps
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS               NAMES
7b0dfb366bcb        ubuntu              "/bin/bash -c 'while…"   10 minutes ago      Up 10 minutes                           zealous_dubinsky
a141b831a16e        ubuntu              "/bin/bash -c 'while…"   25 minutes ago      Up 25 minutes                           ecstatic_pasteur
[root@docker ~]# docker logs -f 7b0dfb366bcb

运行Docker的最佳实践

按用途容器大致可分为两类：服务类容器和工具类的容器。

容器功能分类

  • 服务类容器以 daemon 的形式运行，对外提供服务。比如 web server，数据库等。通过 -d 以后台方式启动这类容器是非常合适的。如果要排查问题，可以通过 exec -it 进入容器。

  • 工具类容器通常给能我们提供一个临时的工作环境，工具类容器多使用基础镜像，例如 busybox、debian、ubuntu 等，通常以 run -it 方式运行。

    [root@docker ~]# docker run -it busybox
    / # ping www.baidu.com
    PING www.baidu.com (14.215.177.39): 56 data bytes
    64 bytes from 14.215.177.39: seq=0 ttl=127 time=5.724 ms
    64 bytes from 14.215.177.39: seq=1 ttl=127 time=4.829 ms

运行 busybox，run -it 的作用是在容器启动后就直接进入。我们这里通过 ping 验证了在容器中访问 internet 的能力。执行 exit 退出终端，同时容器停止。

容器运行小结

容器运行相关的知识点：

  • 当 CMD 或 Entrypoint 或 docker run 命令行指定的命令运行结束时，容器停止。
  • 通过 -d 参数在后台启动容器。
  • 通过 exec -it 可进入容器并执行命令。

指定容器的三种方法：

  • 短ID
  • 长ID
  • 容器名称，可通过 –name 为容器命名

容器按用途可分为两类：

  • 服务类的容器

  • 工具类的容器

start容器

对于处于停止状态的容器，可以通过 docker start 重新启动。

[root@docker ~]# docker ps -a |grep httpd
910bb90c15f5        httpd               "httpd-foreground"       4 minutes ago       Created                                 distracted_kirch
[root@docker ~]# docker start 910bb90c15f5
910bb90c15f5
[root@docker ~]# docker ps -a |grep httpd
910bb90c15f5        httpd               "httpd-foreground"       6 minutes ago       Up 1 second         80/tcp              distracted_kirch

run容器

启动容器包括到容器的创建，然后启动容器两个步骤。
docker run = docker create + dokcer start

[root@docker ~]# docker run -it busybox
/ #

stop容器

容器在 docker host 中实际上是一个进程，docker stop 命令本质上是向该进程发送一个 SIGTERM 信号。

[root@docker ~]# docker ps
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS               NAMES
f55bae074567        busybox             "sh"                     2 hours ago         Up 2 hours                              affectionate_ramanujan
7b0dfb366bcb        ubuntu              "/bin/bash -c 'while…"   2 hours ago         Up 2 hours                              zealous_dubinsky
a141b831a16e        ubuntu              "/bin/bash -c 'while…"   2 hours ago         Up 2 hours                              ecstatic_pasteur
[root@docker ~]# docker stop 7b0dfb366bcb
7b0dfb366bcb
[root@docker ~]# docker ps -a
CONTAINER ID        IMAGE                                                                      COMMAND                  CREATED             STATUS                        PORTS               NAMES
f55bae074567        busybox                                                                    "sh"                     2 hours ago         Up 2 hours                                        affectionate_ramanujan
7b0dfb366bcb        ubuntu                                                                     "/bin/bash -c 'while…"   2 hours ago         Exited (137) 47 seconds ago

kill容器

如果想快速停止容器，可使用 docker kill 命令，其作用是向容器进程发送 SIGKILL 信号。

[root@docker ~]# docker ps -a |grep httpd
910bb90c15f5        httpd               "httpd-foreground"       8 minutes ago       Up About a minute   80/tcp              distracted_kirch
[root@docker ~]# docker kill  910bb90c15f5
910bb90c15f5
[root@docker ~]# docker ps -a |grep httpd
910bb90c15f5        httpd               "httpd-foreground"       8 minutes ago       Exited (137) 5 seconds ago                       distracted_kirch

pause 容器

有时我们只是希望暂时让容器暂停工作一段时间，比如要对容器的文件系统打个快照，或者 dcoker host 需要使用 CPU，这时可以执行 docker pause。

[root@docker ~]# docker ps
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS               NAMES
f55bae074567        busybox             "sh"                     2 hours ago         Up 2 hours                              affectionate_ramanujan
7b0dfb366bcb        ubuntu              "/bin/bash -c 'while…"   2 hours ago         Up 2 minutes                            zealous_dubinsky
a141b831a16e        ubuntu              "/bin/bash -c 'while…"   2 hours ago         Up 2 hours                              ecstatic_pasteur
[root@docker ~]# docker pause f55bae074567
f55bae074567
[root@docker ~]# docker ps -a |grep f55bae074567
f55bae074567        busybox                                                                    "sh"                     2 hours ago         Up 2 hours (Paused)                            affectionate_ramanujan

unpause容器

处于暂停状态的容器不会占用 CPU 资源，直到通过 docker unpause 恢复运行。

[root@docker ~]# docker unpause f55bae074567
f55bae074567
[root@docker ~]# docker ps -a |grep f55bae074567
f55bae074567        busybox                                                                    "sh"                     2 hours ago         Up 2 hours                                     affectionate_ramanujan

rm容器

使用 docker 一段时间后，host 上可能会有大量已经退出了的容器，这些容器依然会占用 host 的文件系统资源，如果确认不会再重启此类容器，可以通过 docker rm 删除。

[root@docker ~]# docker ps -a |grep 'Exited'
e12b964641c4        ubuntu                                                                     "pwd"                    3 hours ago         Exited (0) 3 hours ago                         modest_johnson
97eb5d978d46        ubuntu                                                                     "ls"                     3 hours ago         Exited (0) 3 hours ago

docker rm 一次可以指定多个容器，如果希望批量删除所有已经退出的容器，可以执行如下命令：

docker rm -v $(docker ps -aq -f status=exited)

[root@docker ~]# docker rm -v $(docker ps -aq -f status=exited)
